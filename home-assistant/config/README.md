# Home Assistant Configuration

This Home Assistant instance uses a modular, well-organized configuration structure.

## 📁 Directory Organization

### Configuration Files

- **`configuration.yaml`** - Main configuration with includes
- **`automations/`** - Automation definitions (YAML files)
- **`scripts/`** - Script definitions (YAML files only)
- **`templates/`** - Template sensor/switch definitions
- **`packages/`** - Self-contained feature packages

### Custom Documentation (`docs/`)

All custom documentation created for this configuration:

- **`docs/alexa/`** - Alexa Smart Home integration guides
- **`docs/holidays/`** - Holiday automation system documentation
- **`docs/modes/`** - House mode system documentation
- **`docs/setup/`** - Installation and configuration guides

📖 See [`docs/README.md`](docs/README.md) for complete documentation index.

### Runtime Data (`data/`)

Operational data, logs, and cache files generated by automations:

- **`data/alexa/`** - Alexa-related data files
- **`data/*/`** - Additional service data as needed

💾 See [`data/README.md`](data/README.md) for data directory details.

### Custom Components (`custom_components/`)

Third-party integrations (installed via HACS or manually):

- **`keymaster/`** - Z-Wave lock code management
- **`hacs/`** - Home Assistant Community Store
- **`spook/`** - Enhanced debugging/templating tools

## 🎯 Key Features

### Holiday Automation System

Automatic decoration control based on a comprehensive holiday calendar:

- **Controller:** `automations/holiday_season_controller.yaml`
- **Input Select:** `input_select.active_holiday` (11 holidays)
- **Templates:** `templates/seasonal_*.yaml` (adaptive displays)
- **📚 Docs:** `docs/holidays/HOLIDAY_CALENDAR_REFERENCE.md`

### House Mode System

Comprehensive mode management (Home/Away/Vacation/Guest + Night Mode):

- **Primary Mode:** `input_select.house_mode`
- **Night Mode:** `input_boolean.night_mode`
- **Scripts:** `scripts/good_night.yaml`, `scripts/good_morning.yaml`
- **📚 Docs:** `docs/modes/MODE_AUTOMATION_README.md`

### Alexa Smart Home

Voice control integration with entity filtering:

- **Config:** `alexa.yaml` + `alexa/include/` + `alexa/exclude/`
- **Template Sensors:** Track connection status
- **📚 Docs:** `docs/alexa/ALEXA_QUICK_REFERENCE.md`

### Z-Wave Lock Management (Keymaster)

Advanced lock code management with parent/child sync:

- **Version:** v0.1.1-b1 (with custom Schlage fixes contributed upstream)
- **Locks:** 4x BE469 + 1x FE599 Schlage locks
- **Config:** `custom_components/keymaster/`

## 🗂️ File Organization Principles

### What Goes Where

| Type | Location | Examples |
|------|----------|----------|
| **Config** | `/config/*.yaml` | `configuration.yaml`, `input_select.yaml` |
| **Automations** | `/config/automations/` | `holiday_season_controller.yaml` |
| **Scripts** | `/config/scripts/` | `good_night.yaml`, `good_morning.yaml` |
| **Templates** | `/config/templates/` | `seasonal_displays.yaml` |
| **Docs** | `/config/docs/` | All `.md` files you create |
| **Data/Logs** | `/config/data/` | Runtime files, logs, caches |
| **Custom Components** | `/config/custom_components/` | HACS integrations |

### Source vs. Custom Files

**Upstream/Source (Don't Edit):**

- `.storage/` - HA internal storage (managed by UI)
- `custom_components/*/README.md` - Integration documentation
- `*.log*` - Auto-generated log files

**Custom (Your Work):**

- All files in `docs/` - Your documentation
- All files in `automations/` - Your automations
- All files in `scripts/` - Your scripts
- All files in `templates/` - Your templates
- `configuration.yaml` and related YAML - Your config

## 🔧 Development

### Validating Configuration

```bash
# Via VS Code task
Task: "Check Home Assistant Config"

# Or directly
docker exec home-assistant python -m homeassistant --script check_config -c /config
```

### Restarting Home Assistant

```bash
# Via VS Code task
Task: "Restart Home Assistant"

# Or with Makefile (from repository root)
make restart

# Or directly
docker restart home-assistant
```

### Adding New Documentation

Create files in appropriate `docs/` subdirectory:

```bash
docs/
  ├── alexa/          # Alexa integration docs
  ├── holidays/       # Holiday system docs
  ├── modes/          # Mode system docs
  ├── setup/          # Setup/config docs
  └── [new-topic]/    # Create new subdirectories as needed
```
